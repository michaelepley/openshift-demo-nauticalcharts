Nautical Charts Demo Script

Prepare

  1. Login to Openshift; select one of the following clusters here which known to work with this demo:

  1. Openshift Online
    1. Login to (Openshift Online) using RHN ID
    2. Select pro-us-east-1 zone and click Open Web Console
    3. Create Project (if not already existing) otherwise select nsp-one

    Preconfig

    • Login to GitHub & delete Forked Repo of online_map
    • Delete/scale down all projects in Pro Tier.
    • Delete the PVC and service account for Jenkins

  2. RHSAdemo

    This cluster is maintained by Red Hat Public Sector Middleware Solutions Architects for the purpose of running demos, workshops, training sessions, etc. It is installed on AWS infrastructure, and incorporates a number of other Red Hat Products including Cloudforms for operations support and infrastructure automation, IPA for authentication, and gluster on EC2 for peristent storag capacity. RBAC controls limit standard user accounts to simulate a "normal" developer experience. Openshift system administrator rights are normally not granted to end users.
    1. Login to the rhsademo.net openshift (console) using your login (contact Peter Larsen, plarsen@redhat.com for an account)

    Preconfig

    • Login to GitHub & delete any Forked Repos of online_map
    • If present, delete any conflicting projects, such as username-nauticalcharts

  3. IC Team cluster

    This cluster is maintained by the IC team Solutions Architects for the purpose of running demos, workshops, training sessions, etc. It is installed on hybrid infrasructure on AWS infrastructure and a VRTX located in NicK Sabine's basement, and incorporates a number of other Red Hat Products including Cloudforms for operations support and infrastructure automation, IPA for authentication, and gluster on EC2 for peristent storage capacity.     1. Login to the ocp.rhtps.io openshift console)  using your login (contact Nick Sabine, nsabine@redhat.com for an account)

    Preconfig

    • Login to GitHub & delete any Forked Repos of online_map
    • If present, delete any conflicting projects, such as username-nauticalcharts
  1. Make sure there are no projects existing -- delete any existing projects
  2. Select New Project button to create a new project
  3. Give the project the name nauticalchart-demo

Create an EAP App

We will create a generic EAP application just to show how mutliple apps can co-exist in the same namespace and we can build and deploy complex, java JEE applications.
  1. Click the Add to Project button OR if not available, skip this step
  2. Click the Browse Catalog button
  3. Click the Java family button
  4. Click the Red Hat JBoss EAP button
  5. Click the Select button on JBoss EAP 6.4
  6. Enter an application name: eap-demo
  7. Select the “try it” link and make sure the git URL field is populated.
  8. Explain to application (full JEE application, predating OCP)
  9. Accept the defaults, then scroll down and click Create
  10. Click Continue to Overview
  11. Do NOT wait for the build to complete, proceed with the remaining steps

Create Jenkins Server

Later in this demo, we will

  1. Click the Add to Project  button
  2. Click the Browse Catalog  button
  3. Type “jenkins” in the search field
  4. Click Select on the Jenkins image stream that appears
  5. Accept all defaults, then scroll downand   click Create
    1. Acknowledge the warning and click Create Anyway
  6. Click Continue to Overview
  7. Explain the quota warning (due to PVC of this container) and that we are still good to go
  8. Click Don’t show me again to dismiss the warning

Deploy Original Code

  1. Search in Google for “open nautical chart site:github.com
  2. Look for the following: ‘https://github.com/OpenNauticalChart” - usually the first result
  3. Click the online_map repo
  4. Click the Clone or Download button and copy the link presented (leave the browser tab up)
  5. Return to the OpenShift brower/tab and complete the following:
  1. Click the Add to project button
  2. Click the Browse Catalog button
  3. Click on the PHP Family
  4. Click Select on PHP with version 7 selected
  5. Use these entries for the following fields:
    1. Name: nauticalchart-original
    2. Git Repository URL:<paste the link from step 4>
  1. Click Advanced.  Scroll to bottom of page, change:
    1. Resource Limit  from 512 MiB to  256 MiB
  1. Click Create
  2. Click Continue to Overview
  3. Explain S2I build process.
  4. When Deployment starts, click on POD circle graphic,  explain Details
  5. Switch to Log
  6. Switch to Terminal
  7. Go to the project overview by selecting the Overview button in the upper left corner
  8. Click on nauticalchart-original application route URL.  
  1. Something wrong - advanced capabilities missing.  We need to fix!

Create & Deploy Fork

  1. Return to GitHub
  2. Click the Fork button in the upper right
  1. If you are presented with a choice of multiple accounts, select your personal account.
  2. The fork with take about 10-15 seconds, be patient!
  1. Click the Clone or Download button and copy the link presented in your forked repo
  2. Return to the OpenShift brower/tab and complete the following::
  1. Click Add to project
  2. Click Browse Catalog
  3. Click on the PHP Family
  4. Click Select on PHP with version 7 selected
  5. Use these entries for the following fields:
  1. Name: nauticalchart-forked
  2. Git Repository URL:        <paste the link from step 3>
  1. Click Advanced.  Scroll to bottom of page, change:
  1. Resource Limit from 512 MiB to  256 MiB
  1. click Create
  2. Click Continue to Overview
  3. Wait for build to complete
  4. Show app - still broken, we just forked the same code.
  1. We’re not going to make changes in master, we’re going to create a branch
  2. In GitHub, click the Branch: master button
  3. Type fix in the text field, then click the new button (in blue) to create the fix branch
    1. You will automatically be switched to the fix branch
  4. Back in OpenShift, click Builds on the left menu pane, the Builds in the sub-menu
  5. Click “nauticalchart-forked” in the build list
  6. Click Actions (button in the upper right) and then Edit
  7. Click the link under the Git URL for “advanced options”
  8. Change the Git Reference from “master” to “fix”
  9. Scroll down to the Triggers section and copy the Generic Webhook
  10. Click the Save button at the bottom of the form
  11. On the Build screen, click Start Build to rebuild the app
  12. Return to GitHub, click on the Settings tab at the top of the repo
  13. On the left side, click the Webhooks link
  14. Click the Add Webhook button - you may need to authenticate to GitHub with your password
  15. Paste the Generic Webhook URL into the Payload URL field
  16. Change the Content Type to “application/json”
  17. Click the “Disable SSL” button and acknowledge the warning
    1. Note - you would normally not do this and you can mention that if necessary
  1. Click theAdd Webhook button
  1. Click on/ go to Overview
    1. GitHub will send a test payload to OpenShift to confirm that the webhook works. This should trigger a rolling rebuild of the nauticalchart-forked application. You should see the build number increment by 1 and the rolling deployment update the running app.

And now for something completely different!

  1. Back to EAP! Your EAP app should be completely built by now.
  2. Click the EAP application route URL and you should see the “kitchen sink” app
  3. Test this by adding someone to the list (10 digit phone number, no dashes)
  4. Click on the REST URL to show that it’s working
  5. To reduce resources, spin the app down to 0 by clicking the down arrow next to the pod count - confirm the scale down
  6. Let’s return to the nautical charts

Create Canary Route

  1. Click on the Applications button on the left menu bar then Routes on the sub-menu
  2. Click on the “nauticalchart-original” in the route table
  3. Click Actions in the upper right, then Edit
  4. Check the box for Split Traffic
  5. Select “nauticalchart-forked” from the Service drop-down
  6. Set the Service Weights to 95% of the traffic to original and 5% to forked
  7. Click the Secure Route check box and keep all of the defaults
  8. Click Save at the bottom of the screen

Add Readiness Check

  1. Return to the Overview screen
  2. Back in OpenShift, click Application on the left menu pane, the Deployments in the sub-menu
  3. Click “nauticalchart-forked” in the build list
  4. Click the Actions button in the upper right, then select Edit Health Checks
  5. Click Add Readiness Probe
  6. Change the Type to “Container Command”
  7. In the Command field, type “sh” (without the quotes) then click Add at the end of the line.
  8. In the Add argument field below the sh command, enter “-c” (without the quotes) then click Add at the end of the line.
  9. In the second Add argument field, enter “curl localhost:8080 | grep -i weather” (without the quotes) then click Add at the end of the line.
  10. Click Save at the bottom of the page. Note that this will trigger a rebuild.
  11. Return to the Overview screen and expand the nauticalchart-forked app
  12. Notice that the deployment seems to be stuck. Click View events next to the deployment status
  13. Notice that the readiness probe we just created has failed and the pod is “Unhealthy.” Let’s fix that.
  14. Go back to GitHub - you’re probably still on the webhook screen, so click the online_map link at the top next to your account name - this will return you to the repo - verify the fix branch (change to it if needed)
  15. Click on the index.php file in the content list, this will show you the file contents.
  16. Click the Edit button in the upper right (looks like a pencil)
  17. Near the bottom of the file, change the “<?” to “<?php” (without the quotes)
  1. Add a comment to your change (Fixed missing php, or something like that)
  1. Click Commit Changes
  2. Click the online_map link next to the Branch:fix (NOT the one at the top of the page)
  3. Now select the weather_index.php file from the content list.
  4. Click the Edit button in the upper right (looks like a pencil)
  5. Near the bottom of the file, change the “<?” to “<?php” (without the quotes)
  1. Add a comment to your change (Fixed missing php, or something like that)
  1. Click Commit Changes
  2. Now return to OpenShift and click Overview - you should be in time to see the rebuild to the fixed version of the app.
  1. Explain that this was actually 2 rebuilds - one for each commit in Git - show build numbers
  1. In the app details (below the pod ring) wait for the build to complete then click on the forked URL (NOT original) and wait for the app to load.
  2. You should now have a menu at the top left. Click the View button and enable Weather. This will take several seconds to load, but you should now see a weather overlay.

Now, let’s create a pipeline!

  1. Return to the OpenShift Overview
  2. Go to and copy the contents https://raw.githubusercontent.com/michaelepley/openshift-demo-nauticalcharts/master/resources/build-pipeline-nauticalchart-repair-original.json
  3. Click Add to Project button
  4. Click Import YAML/JSON tab
  5. Paste in the contents of the file
  6. Click Create button at bottom of page
  7. In OpenShift, click Builds on the left menu pane, the Pipelines in the sub-menu
  8. Click “nauticalchart-repair-pipeline” link
  9. Click Actions → Edit
  10. Go to and copy the contents https://raw.githubusercontent.com/michaelepley/openshift-demo-nauticalcharts/master/resources/jenkinsfile-nauticalchart-repair-original.json
  11. Paste into the edit box, replacing All existing contents
  12. Click Save button
  13. Click Start Pipeline button
  14. Return to Overview and expand nauticalchart-forked
  15. Scroll to the bottom and notice the pipeline build working
  16. Once the build enters the Test phase, wait for it to pause for user input, click input required link
  1. Note that this will open the Jenkins page, you may have to accept permissions here.
  1. Once you are presented with the Paused for Input page, select Proceed
  2. Return to the Overview and watch the remainder of the pipeline build.
  3. Notice the following:
  1. The build scales to 2 pods
  2. The build changes the traffic split to 100% forked
  3. The build removes the nauticalchart-original app
  1. Huzzah!
